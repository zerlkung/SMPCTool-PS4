# รายงานการวิเคราะห์และปรับปรุง SMPCTool สำหรับ Spider-Man (PS4)

## บทนำ

เอกสารฉบับนี้สรุปผลการวิเคราะห์ความแตกต่างทางเทคนิคระหว่างไฟล์เกม Spider-Man เวอร์ชัน PC และ PS4 รวมถึงรายละเอียดการปรับปรุงซอร์สโค้ดของโปรแกรม **SMPCTool** (Spider-Man PC Tool) เพื่อให้สามารถรองรับการทำงานกับไฟล์เวอร์ชัน PS4 ได้อย่างสมบูรณ์แบบ โดยการปรับปรุงนี้ออกแบบมาให้โปรแกรมสามารถตรวจจับแพลตฟอร์มได้อัตโนมัติ (Auto-detect) และทำงานร่วมกับไฟล์ทั้งสองเวอร์ชันได้ในโปรแกรมเดียว

## การวิเคราะห์ความแตกต่างระหว่าง PC และ PS4

จากการวิเคราะห์ไฟล์จริงจากเวอร์ชัน PS4 (ไฟล์ `toc`, `dag`, และ `p000045`) เปรียบเทียบกับโครงสร้างเดิมของ PC พบความแตกต่างทางเทคนิคที่สำคัญในหลายส่วน ซึ่งส่งผลโดยตรงต่อการอ่านและเขียนข้อมูล Mod

### 1. โครงสร้างไฟล์ Table of Contents (TOC)

ไฟล์ TOC เป็นไฟล์สารบัญหลักที่เก็บข้อมูลตำแหน่งและขนาดของ Asset ทั้งหมดในเกม แม้ว่าโครงสร้างพื้นฐาน (จำนวน 6 Sections) จะเหมือนกัน แต่มีจุดแตกต่างที่สำคัญดังนี้

| คุณสมบัติ | เวอร์ชัน PC | เวอร์ชัน PS4 |
| :--- | :--- | :--- |
| **Magic Number (Header)** | `0x891FAF77` (175, 18, 175, 119) | `0x77AF12AF` (175, 18, 175, 119)* |
| **การบีบอัด (Compression)** | Zlib (Single block) | Zlib (รองรับ Multi-block / Streaming) |
| **ขนาด Archive Entry** | 72 ไบต์ | **24 ไบต์** |
| **รูปแบบชื่อ Archive** | `g00sXXXX` | ผสมกัน (`g00sXXX`, `p000XXX`, `a00sXXX`) |

*หมายเหตุ: ค่า Magic Number ของ TOC มีค่าไบต์ที่เหมือนกัน แต่การอ่านแบบ Little-endian อาจตีความต่างกันในบางกรณี*

ความแตกต่างที่สำคัญที่สุดคือ **ขนาดของ Archive Entry ใน Section 0** ซึ่งเวอร์ชัน PS4 มีขนาดเพียง 24 ไบต์ต่อรายการ ประกอบด้วยข้อมูล Unknown (2 ไบต์), Install Bucket (1 ไบต์), Unknown (1 ไบต์), Chunkmap (4 ไบต์), Filename (8 ไบต์), และ Tail data (8 ไบต์) ในขณะที่ PC ใช้ขนาด 72 ไบต์

### 2. โครงสร้างไฟล์ Archive (Asset Storage)

ไฟล์ Archive เป็นที่เก็บข้อมูล Asset จริงของเกม (เช่น โมเดล, พื้นผิว, เสียง) ซึ่งมีความแตกต่างอย่างมากระหว่างสองแพลตฟอร์ม

| คุณสมบัติ | เวอร์ชัน PC | เวอร์ชัน PS4 |
| :--- | :--- | :--- |
| **การตั้งชื่อไฟล์** | ขึ้นต้นด้วย `g00s` (เช่น `g00s000`) | ขึ้นต้นด้วย `p` (เช่น `p000045`) |
| **การจัดเก็บข้อมูล (Padding)** | มีระบบ Padding Blocks | **ไม่มี Padding Blocks (Raw Archive)** |
| **Asset Header** | ข้อมูล DAT1 เริ่มต้นทันที | **มี Header 38 ไบต์ ก่อนข้อมูล DAT1** |
| **Magic ของ Asset Header** | `0x44415431` (DAT1) | `0xB5AF20BA` (Type Hash) |

เวอร์ชัน PS4 จัดเก็บข้อมูลในรูปแบบ Raw Archive โดยไม่มีการใช้ Padding Blocks (Magic `1380012868U`) เหมือนใน PC ทำให้ไม่ต้องผ่านกระบวนการ "Convert Padded Asset Archive" ก่อนการ Mod นอกจากนี้ Asset ทุกตัวใน PS4 จะมี Header ขนาด 38 ไบต์นำหน้าข้อมูล DAT1 เสมอ โดย Header นี้ประกอบด้วย Type Hash และ Metadata อื่นๆ

### 3. โครงสร้างไฟล์ DAG (Dependency Graph)

ไฟล์ DAG ใช้เก็บข้อมูลความสัมพันธ์ (Dependencies) ระหว่าง Asset ต่างๆ ในเกม

| คุณสมบัติ | เวอร์ชัน PC | เวอร์ชัน PS4 |
| :--- | :--- | :--- |
| **Magic Number** | `0x891FAF77` | `0x891F77AF` |
| **โครงสร้าง Header** | Magic(4) + Size(4) + Skip(4) + Zlib | Magic(4) + Size(4) + Skip(4) + Zlib |
| **ตำแหน่งเริ่มต้น Zlib** | Offset 12 | Offset 12 |

## รายละเอียดการปรับปรุงซอร์สโค้ด (SMPCTool-PS4)

เพื่อให้ SMPCTool รองรับการทำงานกับไฟล์ PS4 ได้อย่างสมบูรณ์ ได้มีการปรับปรุงซอร์สโค้ดในหลายส่วน โดยยึดหลักการเพิ่มความสามารถ (Extension) โดยไม่ทำลายการทำงานเดิมของ PC

### 1. ระบบ Auto-Detect Platform (`Globals.cs` และ `MainForm.cs`)

ได้เพิ่มตัวแปร `Globals.Platform` (enum `PlatformMode.PC` และ `PlatformMode.PS4`) และสร้างฟังก์ชัน `AutoDetectPlatform` ใน `MainForm.cs` เพื่อตรวจสอบแพลตฟอร์มโดยอัตโนมัติจากชื่อไฟล์ Archive ในโฟลเดอร์ที่ผู้ใช้เลือก

- หากพบไฟล์ที่ขึ้นต้นด้วย `p0` (เช่น `p000045`) ระบบจะตั้งค่าเป็น PS4 Mode
- หากพบเฉพาะไฟล์ที่ขึ้นต้นด้วย `g00s` ระบบจะตั้งค่าเป็น PC Mode

### 2. การแยกวิเคราะห์ TOC (TOC Parsing) (`TOC.cs`)

- **การคลายการบีบอัด (Decompression)**: ปรับปรุงเมธอด `Decompress` ให้ใช้ Streaming Decompression (`ZlibStream.CopyTo`) เป็น Fallback เพื่อแก้ปัญหา "Incomplete Stream" ที่พบในไฟล์ TOC ของ PS4
- **การอ่าน Archive Entries**: ปรับปรุงเมธอด `ParseArchiveFiles` ให้ตรวจสอบ `Globals.Platform` หากเป็น PS4 จะอ่านข้อมูลทีละ 24 ไบต์ต่อรายการแทน 72 ไบต์
- **การบีบอัดกลับ (Compression)**: ปรับปรุงเมธอด `Compress` ให้เขียน Magic Number ที่ถูกต้องตามแพลตฟอร์ม
- **ระบบ Hash**: ทำให้ไฟล์ `AssetHashes.txt` เป็นตัวเลือกเสริม (Optional) เนื่องจาก PS4 อาจไม่มีไฟล์นี้

### 3. การจัดการไฟล์ Archive (`MainForm.cs` และ `InstallModForm.cs`)

- **ข้ามการแปลง Padding Blocks**: ใน PS4 Mode ฟังก์ชัน `convertPaddedAssetArchivesToolStripMenuItem_Click` จะแจ้งเตือนผู้ใช้ว่าไม่จำเป็นต้องแปลงไฟล์ เนื่องจาก PS4 ไม่ใช้ Padding Blocks
- **การดึงข้อมูล Asset**: ปรับปรุงเมธอด `GetAssetBytes` ให้แยกตรรกะเป็น `GetAssetBytesPC` และ `GetAssetBytesPS4` โดยใน PS4 จะอ่านข้อมูลจาก Offset โดยตรงโดยไม่ต้องผ่านกระบวนการ Decompress Padding Block
- **การติดตั้ง Mod**: ใน `InstallModForm.cs` ได้เพิ่มเงื่อนไขสำหรับ PS4 ให้เขียนข้อมูล Mod ลงไปแทนที่ใน Archive โดยตรง (Direct byte replacement) พร้อมปรับปรุง Offset ให้ถูกต้อง

### 4. การจัดการ Texture และ Asset Info (`TextureToolForm.cs` และ `MainForm.cs`)

- **การข้าม PS4 Asset Header**: ปรับปรุงการแสดงผลข้อมูล Asset และระบบ Texture Tool ให้ตรวจสอบและข้าม Header ขนาด 38 ไบต์ของ PS4 (หา Magic `1TAD` ที่ Offset 38) เพื่อให้สามารถอ่านข้อมูล DAT1 และ Texture Metadata ได้อย่างถูกต้องตามโครงสร้างเดิม

### 5. การจัดการไฟล์ DAG (`DAG.cs`)

- ปรับปรุงการตรวจสอบ Magic Number ให้รองรับทั้ง `0x891FAF77` (PC) และ `0x891F77AF` (PS4)
- ปรับปรุงกระบวนการ Decompress ให้รองรับการข้ามไบต์ที่ไม่รู้จักก่อนถึง Zlib Header อย่างถูกต้องสำหรับ PS4

## สรุปผลการดำเนินการ

โปรเจกต์ SMPCTool-PS4 ได้รับการปรับปรุงให้รองรับไฟล์จากแพลตฟอร์ม PS4 อย่างสมบูรณ์ โดยยังคงความสามารถในการทำงานกับไฟล์ PC ไว้เช่นเดิม การปรับปรุงนี้ครอบคลุมตั้งแต่การอ่าน/เขียนไฟล์สารบัญ (TOC), การจัดการกราฟความสัมพันธ์ (DAG), การแยกและติดตั้งข้อมูล (Archive/Mod Installation), ไปจนถึงการจัดการพื้นผิว (Texture Tool) ผู้ใช้สามารถนำโปรแกรมที่ปรับปรุงแล้วนี้ไปใช้ในการสร้างและติดตั้ง Mod สำหรับ Spider-Man เวอร์ชัน PS4 ได้ทันที
